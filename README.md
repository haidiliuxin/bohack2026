# NeuroGarden AI 疗养与家庭守护系统

NeuroGarden 是一个面向 Android 手机端与 Wear OS 手表端的 AI+疗养项目。项目目标不是诊断疾病，而是学习用户自己的日常节律，在心率、呼吸、输入节奏等结构化信号持续偏离时，提供温和提醒、主动陪伴和守护人确认。

> NeuroGarden 不提供医疗诊断，也不能替代医生、心理咨询师或专业机构。系统仅根据用户授权采集的结构化数据，辅助观察当前状态是否持续偏离个人日常节奏，并在需要时提供本地提醒或授权守护确认。

## 当前能力

### 1. 个体化状态监测

项目会围绕“个人基线”判断状态变化，而不是用固定阈值粗暴判定情绪。

当前可参与判断的数据包括：

- 心率 `heartRate`
- 呼吸频率 `breathRate`
- 运动干扰 / 运动强度 `motionLevel`
- 打字速度
- 删除频率
- 输入停顿时长
- 时间段，例如白天、夜间、深夜
- 天气因素，例如天气、温度、湿度、城市
- 个人历史基线与动态阈值
- 用户反馈与守护人反馈

目前不正式采集和入库“当前正在使用哪个 App”的具体名称。无障碍链路中仅用于调试和分类参考，后续如接入也建议只使用“社交 / 短视频 / 游戏 / 学习办公 / 浏览器 / 其他”等应用类别，不采集页面文本、视频标题或聊天内容。

### 2. AI 情绪 / 状态评估

系统会先进行本地风险计算，再将结构化特征交给 Agent 做辅助判断。

Agent 输入边界：

- 允许：心率、呼吸、运动、打字速度、删除频率、停顿、天气、时间段、基线偏离、阈值、最近结构化信号、用户性格/心理模型摘要
- 不允许：输入原文、聊天内容、密码、完整私密文本、未授权健康原始数据

Agent 输出会被规范化为：

- 当前推测状态 / 情绪
- 风险等级
- 置信度
- 主要原因
- 建议动作
- 陪伴话术

当前状态标签不只分为“正常 / 异常”，而是支持更细的表达，例如平静、疲惫、低落、烦躁、焦虑或不安、压力偏高、孤独或空落、积极、中性偏平稳、需要确认的状态波动等。

## 已有成果

### 情绪判断可行性

项目已经完成从“结构化信号”到“情绪 / 状态推测”的基础验证。当前判断不是靠单一指标，例如只看心率或只看打字速度，而是综合：

- 心率、呼吸、运动干扰
- 打字速度、删除频率、停顿时长
- 时间段、天气和场景类别
- 个人基线偏离
- 本地规则与 Agent 结构化分析
- 数据可信度与运动干扰修正

因此，系统更适合判断“状态节律是否偏离”和“可能属于哪一类情绪状态”，而不是直接给用户下医学结论。

### 当前评估结果

项目内置了结构化情绪评估集与回放脚本，位于 `docs/evaluation/`。

在 `neurogarden_emotion_test_100.jsonl` 的 100 条闭集结构化样本上，当前优化版本结果为：

```text
样本数：100
完成数：100
错误数：0
具体情绪严格准确率：66%
情绪大类准确率：96%
输出结构合规率：100%
```

各标签表现存在差异：

```text
专注、积极活跃、运动干扰：100%
烦躁、焦虑、疲惫：约 70%
平静：约 60%
紧张、低落、空落：仍需继续优化
```

在 4000 条覆盖样本的泛化测试中，项目记录的结果为：

```text
具体情绪泛化准确率：约 62.35%
情绪大类准确率：约 80.7%
```

这些结果说明：当前系统已经能较稳定判断“状态方向”和“情绪大类”，并能在部分高特征差异情绪上达到可演示水平。但它仍不是临床准确率，也不应被宣传为医学诊断能力。下一步重点是提升紧张、低落、空落等边界情绪的细粒度准确率。

### 已实现闭环

NeuroGarden 目前已经实现一条可演示的状态守护闭环：

```text
用户授权权限
-> 后台采集心率 / 呼吸 / 运动 / 输入节奏
-> 清洗与数据可信度评估
-> 学习个人基线
-> 本地风险计算
-> Agent 结构化情绪判断
-> 生成 risk_events 风险事件
-> 今日页展示当前状态、曲线和原因
-> 异常时触发 App 内弹窗 / 悬浮窗 / 通知
-> 用户进入话聊助手或呼吸引导
-> 守护页支持短信草稿通知家人
-> 用户 / 守护人反馈
-> FeedbackTuningEngine 调整 threshold_profiles
-> 后续判断受反馈影响
```

这意味着项目已经不只是一个静态 UI Demo，而是具备“采集、判断、解释、提醒、陪伴、反馈、调参”的完整原型链路。

### 3. 四栏主界面

手机端主界面包含：

- 今日：状态摘要、曲线、当前推测情绪、风险事件
- 守护：家庭守护、紧急联系人、个人病例、短信草稿、反馈
- 话聊助手：主动陪伴、轻柔追问、呼吸 / 专注模式
- 设置：个人资料、权限、模式切换、开发者调试

### 4. 三种使用模式

#### 自我监测

- 只做本地观察和温和提醒
- 不通知守护人
- 适合普通用户日常自我了解

#### 家庭守护

- 支持紧急联系人
- 支持守护提醒记录
- 支持短信草稿通知
- 支持用户 / 守护人反馈
- 用于家人之间的轻量确认

#### 特殊关怀

- 更关注连续偏离、夜间异常、高心率低运动、多指标同步偏离
- 提醒冷却更严格，避免频繁打扰
- 更适合需要长期陪伴或照护的人群

## 守护闭环

当前核心闭环为：

```text
权限配置
-> 数据采集
-> 个人基线学习
-> 本地风险计算
-> Agent 结构化分析
-> 风险事件落库
-> 今日 / 历史页面展示
-> 后台通知 / 悬浮窗提醒
-> 用户 / 守护人反馈
-> threshold_profiles 阈值调参
-> 后续风险判断受反馈影响
```

## 被动监测与提醒

### 后台守护服务

- 使用前台服务运行
- 周期性读取输入节奏与手表 / 模拟手表数据
- 计算交互风险、生理风险和综合风险
- 结合数据可信度判断是否提醒
- 生成 `risk_events` 风险事件

### 悬浮窗与 App 内弹窗

- 如果用户开启悬浮窗权限，可以在 App 后台显示关怀提醒
- 没有悬浮窗权限时，可退回通知栏或 App 内待处理弹窗
- 弹窗支持进入话聊助手
- 悬浮窗中的“呼吸引导”会跳转到话聊助手右上角对应的呼吸 / 专注模式

### 开发者调试

设置页的“开发者调试”提供：

- 切换模拟数据
- 导入演示数据
- 打开无障碍设置
- 打开蓝牙设置
- 打开悬浮窗权限
- 查看 Debug 日志
- 呼出应用内弹窗
- 延迟 1 分钟呼出悬浮窗，便于路演展示
- 连接手表
- 调整模拟心率 / 呼吸
- 清除本地习惯记忆

## 短信通知能力

当前短信方案采用“打开系统短信草稿 + 用户确认发送”，不是后台自动发送短信。

流程：

1. 用户在守护页点击“短信通知”
2. App 自动打开系统短信应用
3. 收件人填入紧急联系人手机号
4. 正文自动生成温和模板
5. 用户自行确认是否发送

短信模板会包含：

- 当前推测状态 / 情绪
- 触发原因
- 给守护人的温和建议
- 隐私说明和非医疗诊断声明

示例：

```text
【NeuroGarden】刚刚观察到 TA 的状态节律有些偏离日常，建议你方便时轻轻确认一下。当前推测状态：紧张或压力偏高。原因：呼吸频率高于个人基线、打字速度偏离个人习惯。可以先问：“我在，今天还好吗？” 本提醒不含聊天原文，仅用于守护确认，不代表医疗诊断。
```

这种方案比申请 `SEND_SMS` 自动发短信更合规，也更适合比赛演示和真实用户授权边界。

## 数据与隐私边界

NeuroGarden 不保存：

- 用户输入原文
- 聊天内容
- 密码内容
- 原始私密文本
- 完整敏感对话
- 当前页面文本、视频标题、聊天对象等内容

NeuroGarden 当前保存或使用：

- 心率
- 呼吸频率
- 运动状态
- 打字速度
- 删除频率
- 停顿时长
- 天气
- 时间段
- 风险评分
- 当前推测状态
- 异常事件摘要
- 用户 / 守护人反馈

## 手动验收建议

1. 打开设置页，确认通知、无障碍、悬浮窗、蓝牙入口可进入系统设置
2. 在开发者调试中开启模拟手表数据
3. 导入演示数据，观察今日页曲线和风险事件
4. 进入守护页，编辑紧急联系人和个人病例
5. 点击短信通知，确认系统短信草稿可以打开
6. 点击呼出应用内弹窗，确认能进入话聊助手
7. 点击延迟呼出悬浮窗，等待 1 分钟确认后台悬浮窗展示
8. 在悬浮窗中点击呼吸引导，确认跳转到话聊助手呼吸 / 专注模式

## 当前限制

- 情绪判断是“概率性状态推测”，不是准确诊断
- 真实 Wear OS Health Services 接入仍需继续完善和适配设备
- 短信采用系统草稿方式，不做后台自动发送
- 当前 App 名称 / 当前浏览内容没有正式采集入库
- Agent 依赖网络和 API 可用性，失败时会走 fallback
- 数据可信度较低时，应以记录和观察为主，不应强提醒

## 实用价值

NeuroGarden 的实用价值在于把“用户主动记录心情”转变为“系统在用户允许范围内被动观察状态节律”，特别适合以下场景：

- 年轻人压力管理：在学习、工作、社交压力升高时，提供低打扰提醒和呼吸引导
- 家庭守护：让家人获得经过用户确认的状态提示，而不是窥探隐私内容
- 老人陪伴：在长期缺乏陪伴或作息异常时，提供轻量关怀入口
- 康养辅助：帮助用户观察睡眠、压力、输入节奏和身体信号之间的关系
- 赛事演示和科研原型：提供 Android + Wear OS + LLM 的端到端可验证系统

项目的核心价值不是替用户判断“你一定怎么了”，而是在状态持续偏离时更早、更温和地递出一个支持入口。

## 商业价值

NeuroGarden 的商业方向更适合做“授权式状态守护基础设施”，而不是售卖用户情绪数据。

潜在方向包括：

- To C 订阅：个人压力管理、AI 陪伴、睡眠与状态节律报告
- 家庭守护服务：为老人、学生、异地家人提供轻量状态确认
- 智能手表 / 手机生态合作：作为 Wear OS 与 Android 健康数据的情绪状态解释层
- 康养机构工具：为疗养、康复、养老机构提供非诊断型状态观察面板
- 企业员工关怀：只提供聚合和匿名趋势，避免个人监控
- AI Agent 能力接口：向其他应用提供“用户授权后的状态上下文”，让提醒、陪伴、内容推荐更有边界和温度

商业化必须坚持三条边界：

- 不出售原始输入内容
- 不向第三方转发未经授权的个人情绪细节
- 不宣称医疗诊断、疾病预测或危机预测

在这个边界内，NeuroGarden 可以成为一个可信的 AI+疗养入口：既能展示技术能力，也能避免落入“监控软件”的风险。

## 项目定位

NeuroGarden 面向 BoHack “AI+疗养”主题，强调个体化、温和、可解释、可反馈的状态守护。它试图连接手机和手表端的结构化信号、AI 的状态理解能力、用户自己的反馈，以及家人/照护者的轻量确认，形成一个尊重隐私边界的康养辅助闭环。
