# NeuroGarden 多日验收测试数据库方案（待审核）

这份数据用于手动验收“今日、历史、守护、设置、风险事件、天气因素、数据可信度、监护反馈”等功能是否可用。当前只生成审核文件和 SQL 脚本，尚未写入 App 数据库。

配套 SQL 文件：

- `docs/neurogarden_acceptance_seed.sql`

## 覆盖范围

- 用户数量：1 个本地测试用户，不包含姓名、手机号等真实身份信息。
- 时间跨度：2026-05-17 到 2026-05-23，共 7 天。
- 每天样本：每天 3 个检测点，覆盖上午、下午、夜间。
- 样本字段：心率、呼吸频率、运动状态、打字速度、删除频率、停顿时长。
- 风险事件：5 条，覆盖轻度波动、夜间异常、运动干扰误报、家庭守护确认、特殊关怀继续观察。
- 反馈记录：4 条，覆盖“有点累”“需要陪伴”“误报”“提醒有效”。
- 天气因素：风险事件中包含天气、温度、湿度、城市和 Real/Mock 来源。

## 设计意图

| 日期 | 场景 | 验收重点 |
| --- | --- | --- |
| 2026-05-17 | 稳定日 | 历史页稳定数据、个人基线 |
| 2026-05-18 | 轻度波动 | 今日摘要、observe 事件、不通知监护人 |
| 2026-05-19 | 恢复稳定 | 7 天趋势回落 |
| 2026-05-20 | 夜间异常 | guardian_check、天气、已联系本人反馈 |
| 2026-05-21 | 运动干扰 | 高心率但 motionLevel 高，验证误报/置信度下降 |
| 2026-05-22 | 家庭守护 | 监护人确认、反馈调参入口 |
| 2026-05-23 | 特殊关怀 | 高敏感照护提醒、继续观察反馈 |

## 关键基线

- 平均静息心率：72 bpm
- 平均呼吸频率：12.5 次/分钟
- 平均打字速度：108 字/分钟
- 平均删除频率：0.05
- 平均停顿时长：1.6 秒
- 基线可信度：high

## 验收时应该看到

- 历史页能看到 7 天内多条事件和摘要。
- 今日页能展示天气因素、状态评分、异常事件。
- 异常详情页能展示相对个人基线偏离、天气、时间段和反馈按钮。
- 数据可信度不应一直为 low，因为种子数据包含心率、呼吸、输入节奏、天气和 Wear/手表语义标签。
- 运动干扰事件应带有误报标记，便于展示反馈校准。
- 家庭守护和特殊关怀场景能展示不同提醒文案和反馈状态。

## 审核后导入方式

审核通过后，可以选择两种方式之一：

1. 我把它接入 App 的 Demo Mode，增加一个“导入多日验收数据”按钮。
2. 你在测试机或模拟器的 `neurogarden.db` 里执行 `docs/neurogarden_acceptance_seed.sql`。

建议优先选第 1 种，因为更适合比赛现场演示，也不需要手动操作 SQLite。
