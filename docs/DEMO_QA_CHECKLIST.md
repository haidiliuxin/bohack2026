# NeuroGarden 比赛演示与稳定性检查清单

## 构建检查

在项目根目录执行：

```bash
./gradlew :app:assembleDebug
./gradlew :wear:assembleDebug
```

当前 Windows 本地没有 `gradlew` 文件时，可使用本机 Gradle wrapper 缓存中的 `gradle.bat` 执行同等任务。

成功产物：

- 手机端 APK：`app/build/outputs/apk/debug/app-debug.apk`
- 手表端 APK：`wear/build/outputs/apk/debug/wear-debug.apk`

## 手机端演示流程

1. 打开 NeuroGarden。
2. 进入“设置”，确认当前使用模式。
3. 打开“隐私与权限”，说明系统不保存输入原文、不做医疗诊断。
4. 在“输入节奏采集状态”中展示无障碍服务状态、今日样本数和最近采集时间。
5. 使用 Demo Mode 写入“模拟轻度波动”或“模拟夜间异常”。
6. 回到“今日”，展示今日摘要、天气因素、状态评分和异常事件列表。
7. 点开异常事件，展示结构化原因、相对个人基线偏离、天气和反馈入口。
8. 进入“守护”，切换三种模式，说明通知阈值、每日提醒上限和隐私级别。
9. 进入“历史”，展示 7 天摘要和反馈统计。

## Wear OS 演示流程

1. 在 Android Studio 运行 `wear` 模块到 Wear OS 模拟器或真机。
2. 打开手表端页面。
3. 点击手动采集或模拟心率按钮。
4. 手机端接收 Wear Data Layer 数据后，今日页“最近心率来源”应显示 Real。
5. 若没有手表或连接失败，可以保留 Mock 心率演示，并说明真实接收链路已保留。

## 被动守护演示流程

1. 设置页打开系统无障碍设置。
2. 手动开启“NeuroGarden 输入节奏特征”。
3. 回到 App，启动被动守护。
4. 通知栏应出现 NeuroGarden 前台服务通知。
5. 使用手机输入一段文字后回到设置页，查看今日输入节奏样本数和最近采集时间。
6. 当输入节奏和手表/模拟生理信号同时偏离，系统会按当前使用模式显示温和提醒。

## 稳定性检查

- 网络不可用时，天气显示 Mock，不崩溃。
- 无手表时，系统继续使用输入节奏和 Mock/缺省生理数据，不崩溃。
- 数据可信度为 low 时，只记录状态，不升级为强提醒。
- 自我监测模式不会产生监护人提醒。
- 家庭守护模式每日最多 3 次守护提醒。
- 特殊关怀模式每日最多 2 次照护提醒，且通知冷却更严格。
- 无障碍服务关闭时，设置页能明确显示未开启，并可跳转系统设置。

## 答辩表述

NeuroGarden 不是监控软件，也不是医疗诊断软件。它只处理用户授权的结构化特征，学习个人日常节奏，在状态持续偏离时提供温和、可解释、可反馈的 AI+疗养支持。
