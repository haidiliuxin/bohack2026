# NeuroGarden 前端界面策划案

## 1. 设计定位

NeuroGarden 手机端主界面参考苹果健康、运动、正念类 App 的体验方向：轻量、干净、温和，但信息密度足够支撑日常守护。

整体不是“医疗诊断工具”，而是一个面向个人和家庭的情绪与身体节律守护 App。视觉上应避免沉重、病理化、监控感过强的表达，优先呈现“今天过得怎么样”“身体节律是否稳定”“需要一点陪伴吗”。

核心气质：

- 白色或极浅灰背景，搭配柔和蓝色、蓝绿色、珊瑚红、薄荷绿等健康类色块。
- 卡片圆角克制，信息卡片清楚分组，不做复杂装饰。
- 底部 4 个 Tab 使用毛玻璃样式，图标为蓝色高亮，整体参考 iOS Health / Fitness 的轻盈感。
- 图标风格统一为线性或轻拟物，不使用厚重插画。
- 文案温和，不诊断，不吓人。

## 2. 全局导航

底部固定 4 个 Tab：

| Tab | 标题 | 图标 | 主要任务 |
|---|---|---|---|
| 1 | 今日 | 星星 | 查看今日身体与情绪状态、今日评分、历史趋势、预警记录 |
| 2 | 守护 | 心形镂空盾牌 | 调整守护阈值、查看运行状态、管理紧急联系人和个人简洁病例 |
| 3 | 话聊 | 聊天气泡 | AI 陪伴对话、正念呼吸、情绪粒子球反馈 |
| 4 | 设置 | 设置 | 个人信息、使用模式、开发调试入口 |

底部 Tab 样式：

- 背景：半透明白色毛玻璃，带轻微 blur 和浅阴影。
- 激活态：蓝色图标 + 蓝色文字，可增加轻浅圆形底。
- 非激活态：灰蓝色图标 + 灰色文字。
- 形态：悬浮于底部安全区上方，iOS 风格圆角胶囊容器。

## 3. Tab 1：今日

### 3.1 页面目标

让用户一打开 App 就知道：

- 今天身体节律大致如何。
- 当前设备是否连接。
- 今日心情/状态评分如何。
- 最近一周趋势是否变好。
- 今天是否出现过需要注意的预警。

### 3.2 页面结构

从上到下：

1. 顶部标题区
2. 四宫格指标入口
3. 今日评分卡片
4. 历史评分卡片
5. 历史预警记录列表

### 3.3 顶部标题区

标题栏居左，显示问候语：

- `你好，林林`
- `今天感觉怎么样？`
- `早上好，林林`
- `晚上好，先慢下来一点`

标题下方可以显示一行轻提示：

- `今日守护已运行 6 小时 24 分`
- `手表数据同步于 2 分钟前`

### 3.4 四宫格大色块

参考苹果健康分类卡片。两列布局，每个卡片为大色块，包含标题、实时值/状态、更新时间和简单图形。

心率、呼吸、体温必须在首页直接展示实时数据，而不是只作为入口卡片。实时数据来自手表、手机传感器或后端聚合结果；当数据源暂时不可用时，卡片应显示缺失态和最近一次有效时间。

四个卡片：

| 卡片 | 实时内容 | 建议颜色 | 交互 |
|---|---|---|---|
| 心率 | 实时 BPM、趋势箭头、更新时间，例如 `78 BPM · 刚刚` | 珊瑚红 / 粉红 | 点击进入心率详情 |
| 呼吸 | 实时每分钟呼吸次数、平稳程度、更新时间，例如 `16 次/分 · 平稳` | 薄荷绿 / 青绿 | 点击进入呼吸详情 |
| 体温 | 实时体温或皮肤温度、偏离程度、更新时间，例如 `36.6°C · 正常` | 浅橙 / 米杏 | 点击进入体温详情 |
| 设备 | 手表连接状态、电量、同步时间 | 天蓝 / 靛蓝 | 点击打开蓝牙/设备连接界面 |

实时数据展示规则：

- 主数值必须大于说明文字，用户扫一眼能看到当前状态。
- 数据新鲜度超过阈值时显示 `5 分钟前`、`同步延迟` 等提示，不继续假装实时。
- 数据缺失时显示 `暂无数据`，并给出轻操作提示，例如 `检查手表连接`。
- 趋势箭头只表达相对个人基线的轻微变化，不直接做疾病判断。
- 调试阶段可由开发者调试入口注入模拟心率、呼吸、体温数据，用于验证首页状态。

设备卡片要求：

- 未连接：显示 `未连接`、蓝牙图标、`连接手表`
- 已连接：显示设备名、电量、最近同步时间
- 点击后进入设备页或系统蓝牙入口，并提供 Wear OS 手表连接引导

### 3.5 今日评分卡片

卡片内容：

- 左侧：大 emoji 或轻插画情绪符号，尺寸应明显大于普通文本图标
- 右侧：数字评分
- 数字下方：一句话今日评价

评分建议使用 0-100 分，数字足够大，但不制造考试感。

情绪符号必须根据今日心情评分变化，而不是固定显示一个图标。前端根据评分段选择表情；后端如果额外返回情绪标签，可在同一分数段内微调表情。

评分段表情映射：

| 评分段 | 状态 | Emoji/图形 | 色彩倾向 | 文案例子 |
|---|---|---|---|---|
| 85-100 | 积极 | 阳光微笑人脸，脸部周围带一圈太阳光芒 | 金黄 / 暖橙 | `今天节律不错，继续保持。` |
| 70-84 | 温和 | 柔和微笑人脸，眼神放松 | 浅蓝 / 薄荷绿 | `整体还算平稳，适合慢慢推进今天。` |
| 55-69 | 有点疲惫 | 轻微疲惫人脸，眼睛半闭但不夸张 | 蓝灰 / 淡紫蓝 | `今天有点累，可以把节奏放慢一些。` |
| 40-54 | 低落 | 嘴角向下的人脸，表情委屈或难过 | 冷蓝 / 灰蓝 | `今天有点低落，要不要留一点空间给自己？` |
| 20-39 | 吃力 | 绿脸闭眼、轻微不适感 | 黄绿 / 暗青 | `状态有些吃力，需要我陪你缓一会儿吗？` |
| 0-19 | 高压 | 深呼吸中的紧张脸或痛苦脸，避免恐怖化 | 暗红 / 暖灰 | `现在可能很难受，我们先把呼吸放慢一点。` |

表情设计要求：

- 表情在卡片里应是大号视觉主角，约占卡片高度的 50%-65%。
- 需要可做成统一插画资产或 emoji-like 组件，保持同一光照、描边和圆润程度。
- 低分表情不能做成惊吓、警报或羞辱感，重点是“被看见”和“可被安抚”。
- 首页只展示生活化评价；专业解释仍放在 `守护` 页。

首页文案要偏生活化，不做专业解释。专业解释放在守护页。

### 3.6 历史评分卡片

内容：

- 标题：`历史评分`
- 副标题：`近 7 天`
- 折线图：显示一周分数
- 支持左右滑动切换周
- 可以显示当前周平均分、最高/最低点

图表要求：

- 线条柔和，曲线不要太尖锐。
- 分数点可点击，显示当天简短摘要。
- 不把低分做成红色警报，除非达到明确预警级别。

### 3.7 历史预警记录

在历史评分卡片下方显示 3-5 条最近预警。

列表字段：

- 时间
- 风险级别/提醒类型
- 简短原因
- 当前处理状态

示例：

- `21:32 情绪波动：打字节奏明显变慢，已建议呼吸练习`
- `18:10 轻度紧张：心率和删除率同时升高`
- `昨晚 23:48 睡前波动：连续使用手机时间较长`

点击进入预警详情。

## 4. Tab 2：守护

### 4.1 页面目标

守护页面向更“管理”和“解释”的场景，重点是：

- 调整守护敏感度。
- 查看守护系统是否运行。
- 查看今日提醒次数。
- 获取更专业的 AI 状态评语。
- 管理紧急联系人和个人简洁病例。

### 4.2 页面结构

从上到下：

1. 顶部标题：`守护`
2. 阈值调节大卡片
3. 运行状态卡片
4. 紧急联系人列表项
5. 个人简洁病例列表项

### 4.3 阈值调节卡片

大卡片标题可为 `守护阈值` 或 `敏感程度`。

滑动条建议：

| 滑动条 | 含义 |
|---|---|
| 情绪波动敏感度 | 决定情绪状态偏离基线时多快提醒 |
| 身体信号敏感度 | 心率、呼吸、体温等偏离的触发强度 |
| 输入节奏敏感度 | 删除率、停顿、输入速度异常的权重 |
| 提醒频率上限 | 控制每天最多主动提醒次数 |

交互说明：

- 滑动条左侧是更安静，右侧是更敏感。
- 显示实时解释文案，例如 `当前：适中，适合日常守护`。
- 高敏感模式要提醒可能增加打扰。
- 调整后写入本地阈值配置，并影响后端/算法策略。

### 4.4 运行状态卡片

标题居左：`运行状态`

卡片内部左右分区：

- 左侧小框：今日提醒次数，可点击呼出今日提醒记录。
- 右侧专业评语：由 AI 根据结构化数据生成，比首页更具体。

示例专业评语：

`今天下午出现过一次短暂高唤醒状态，主要来自心率偏高和输入停顿增加。当前已回到个人常规区间，建议晚间降低主动提醒强度。`

注意：

- 这里可以出现“高唤醒”“偏离个人基线”“输入节奏”等专业词。
- 仍然不能说“焦虑症”“抑郁症”等诊断结论。

### 4.5 列表项

紧急联系人：

- 展示联系人数量和最近一次确认状态。
- 点击进入联系人管理。
- 后续可支持通知策略：仅高风险提醒、每日摘要、完全关闭。

个人简洁病例：

- 展示用户主动填写的简洁健康信息。
- 用于紧急联系人或特殊关怀模式下辅助理解。
- 默认不主动上传，不做诊断。

## 5. Tab 3：话聊

### 5.1 页面目标

话聊是 AI 陪伴和正念入口。整体参考 ChatGPT 移动端界面，但更轻、更空、更有“呼吸感”。

### 5.2 常规聊天界面

顶部结构：

- 左上：两个横杠图标，点击打开历史记录。
- 中间胶囊标题：`话聊助手`
- 右上：正念按钮，参考图 3 的圆角方形与同心圆构图，但改为黑白线性/单色样式，避免直接复用原图。

中部：

- 大面积留白。
- 中央放一个由许多粒子组成的球体。
- 球体缓慢旋转，并做呼吸式缩放。

底部：

- 输入框，参考 ChatGPT 样式。
- 中间输入占位：`说说现在的感觉`
- 右侧仅保留发送按钮，蓝色圆形底，图标使用回车/箭头发送符号。
- 不显示 `+`、麦克风、语音波形或任何语音输入控件。
- 该 AI 入口只承载文字对话功能。

### 5.3 粒子球状态系统

粒子球是话聊页的核心视觉资产。

基础状态：

- 颜色：淡蓝到蓝绿色混合。
- 动画：缓慢旋转，平稳呼吸缩放。
- 粒子：细小发光点组成球体，边缘柔和。

根据情绪评估变化：

| 状态 | 颜色 | 呼吸速度 | 缩放幅度 | 旋转 |
|---|---|---|---|---|
| 平稳 | 淡蓝/蓝绿 | 慢 | 中等 | 稳定 |
| 积极 | 蓝绿偏亮，可带一点金色光 | 稍快但顺滑 | 轻微扩大 | 更轻盈 |
| 低落 | 冷蓝/淡紫蓝 | 更慢 | 更小 | 缓慢 |
| 紧张 | 红/粉红/橙红 | 更急促 | 幅度更明显 | 轻微抖动或粒子收缩 |
| 痛苦 | 深红或暗蓝混合 | 不规则但不能刺眼 | 收缩明显 | 较慢、沉重 |

动画必须温和，不能像警报灯。

### 5.4 正念模式

点击右上正念按钮后进入沉浸模式：

- 顶部、历史、输入框、Tab 等界面元素淡出。
- 屏幕仅保留粒子球。
- 底部出现一句引导文案。
- 粒子球进入明确的呼吸节奏动画。

正念引导文案示例：

- `吸气，跟着光慢慢变大`
- `停一下，让身体知道你在这里`
- `呼气，慢慢放下`
- `再来一次，不需要做得很完美`

交互：

- 点击屏幕或左上返回退出正念模式。
- 可配置 30 秒、1 分钟、3 分钟。
- 结束后给一句轻反馈：`刚刚这半分钟，已经很好了。`

## 6. Tab 4：设置

### 6.1 页面目标

设置页不做复杂仪表盘，只承载身份、模式、权限和开发阶段调试。

### 6.2 页面结构

从上到下：

1. 个人信息卡片
2. 使用模式卡片
3. 数据与权限入口
4. 开发者调试入口

### 6.3 个人信息卡片

内容：

- 头像
- 昵称
- UID
- 当前设备绑定状态

示例：

- 昵称：`林林`
- UID：`NG-2048-0913`
- 状态：`已连接 Watch`

### 6.4 使用模式卡片

使用模式：

- 自我检测
- 家庭守护
- 特殊关怀

卡片内使用分段控件或列表选择。每个模式显示一句说明：

- 自我检测：`提醒更克制，数据主要用于自己查看。`
- 家庭守护：`在明显波动时可通知守护联系人。`
- 特殊关怀：`提醒更敏感，但每日打扰次数受限。`

### 6.5 开发者调试入口

开发阶段保留一行：

`开发者调试`

点击后进入调试页面，放置：

- 模拟心率、呼吸、体温输入
- 模拟情绪状态
- 打开无障碍设置
- 打开蓝牙设置
- 打开悬浮窗权限
- 查看采集 Debug 日志
- 导入演示数据
- 触发测试预警
- 清空本地习惯记忆

发布前处理：

- 直接移除该入口，或仅在 debug build 显示。
- 不影响正式用户路径。

## 7. 数据与后端依赖

首页需要：

- 实时心率、实时呼吸、实时体温/皮肤温度、设备状态
- 各实时数据的更新时间、数据来源和缺失状态
- 今日评分
- 今日评分对应的表情段和可选情绪标签
- 今日一句话评价
- 近 7 天评分序列
- 最近预警记录

守护页需要：

- 当前阈值配置
- 今日提醒次数
- 今日提醒记录
- AI 专业状态评语
- 紧急联系人
- 个人简洁病例

话聊页需要：

- 当前情绪评估结果
- 粒子球状态参数：颜色、呼吸速度、缩放幅度、旋转速度
- 对话历史
- 正念模式状态
- AI 陪伴回复

设置页需要：

- 用户资料
- UID
- 当前使用模式
- 权限状态
- 开发调试开关

## 8. 文案原则

首页文案：

- 更生活化。
- 少专业术语。
- 像一个温柔的日常提醒。

守护页文案：

- 更专业。
- 可以解释信号来源。
- 仍然不能做医疗诊断。

话聊文案：

- 不说教。
- 每次最多问一个轻问题。
- 高压状态下先稳定，再解释。

设置文案：

- 清楚、短、可操作。

## 9. 视觉预览生成说明

本策划案配套生成 4 张竖屏 UI 预览图：

1. `今日`：苹果健康风格四宫格 + 今日评分 + 历史趋势 + 预警记录。
2. `守护`：阈值滑动条 + 运行状态 + 联系人与病例入口。
3. `话聊`：ChatGPT 风格顶部与输入框 + 中央粒子球 + 正念入口。
4. `设置`：个人信息 + 使用模式 + 开发者调试入口。

预览图仅用于对齐视觉方向，不作为最终可实现稿。后续落地时应使用 Jetpack Compose 原生组件、Canvas 或粒子动画实现，而不是直接切图。

## 10. 推荐落地顺序

1. 先改底部 4 Tab 和基础页面骨架。
2. 完成 `今日` 页四宫格、今日评分、历史折线和预警列表。
3. 完成 `守护` 页阈值卡片和运行状态卡片。
4. 完成 `设置` 页个人信息、使用模式和 debug-only 调试入口。
5. 最后实现 `话聊` 页粒子球和正念沉浸模式，因为它涉及动画和状态映射，风险最高。

## 11. 前端实现备注

- 项目当前为 Android Kotlin + Jetpack Compose，应优先用 Compose 原生实现。
- 底部毛玻璃可用半透明 Surface、blur、阴影和 navigationBarsPadding 组合模拟。
- 折线图可继续使用 Canvas，但需要做成独立组件。
- 粒子球建议用 Canvas 自绘，先做 2D 伪 3D 粒子球，再考虑更复杂方案。
- 图标建议使用 Material Icons 或项目内统一矢量资源；守护盾牌和正念图标可定制 VectorDrawable。
- 开发者调试入口必须通过 build type 或 feature flag 控制，避免发布残留。
