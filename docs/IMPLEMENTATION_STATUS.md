# NeuroGarden 当前实现状态

## 已完成

### 情绪判断准确性

- 个人习惯样本表、个人基线表、阈值配置表。
- 低负担反馈表。
- 多维情绪状态估计：高压紧张、焦虑紧绷、低落疲惫、疲惫恢复慢、运动干扰、相对稳定。
- 趋势窗口：5 分钟、30 分钟、2 小时。
- 有效提醒率统计。
- 用户轻量标注：累、烦、低落、紧张、没事。
- 近期反馈校准：误报多时降低打扰强度，有效反馈多时提升置信度。

### 被动守护

- 手机端前台服务。
- 本地高风险通知。
- 通知冷却：30 分钟内不重复提醒。
- 每日通知上限：4 次。
- 无障碍输入节奏特征采集：只统计打字速度、删除频率、停顿时长。

### AI 关怀

- 后端大模型 API 接入。
- Mock fallback。
- 异常后轻柔对话。
- 对话中二次风险判断。
- 对话摘要入库，不保存原始敏感全文。

### 多元用户支持

- 守护联系人模拟通知。
- 简化关怀模式。
- 隐私与权限说明页。
- 本地习惯记忆清除。

### Wear OS

- Wear 端 Mock 心率。
- Wear 数据 payload 构造。
- 手机端 Wear payload 解析。
- 手机端真实 Wear packet 进入统一风险判断入口。
- Health Services 真实接入状态与权限预留。

## 仍需真实接入

### 真实 Wear OS 被动监听

当前还没有完整实现 Health Services Passive Monitoring。下一步需要：

- 使用 Health Services `PassiveMonitoringClient` 或合适的 Measure API。
- 处理心率权限和后台传感器权限。
- 获取真实心率、运动状态、活动类型。
- 通过 Wearable Data Layer 发送到手机。

### 真实通知守护人

当前“通知守护人”仍是模拟。真实实现需要：

- 联系人授权。
- 二次确认。
- 服务端通知或短信能力。
- 通知记录。
- 撤销授权。

### 数据删除细化

当前可清除本地习惯记忆。后续可以拆成：

- 清除关怀记录。
- 清除习惯样本。
- 清除 AI 对话摘要。
- 重置阈值。

### 历史趋势可视化

当前历史页只有摘要。后续可以增加：

- 风险趋势图。
- 高风险时段统计。
- 标注分布。
- 有效提醒率变化。
- 阈值调整记录。

## 比赛演示建议

1. 启动守护模式。
2. 打开隐私与权限说明，说明系统不采集文本、不做诊断。
3. 进入模拟体验，切换到焦虑或运动干扰。
4. 展示可能状态、趋势判断、置信度和解释。
5. 进入守护关怀，展示轻柔提问。
6. 标注“低落”或“紧张”，展示系统校准。
7. 打开历史页，展示关怀记忆摘要和有效提醒率。
8. 开启简化关怀模式，展示老人/疲惫用户友好的低交互界面。
