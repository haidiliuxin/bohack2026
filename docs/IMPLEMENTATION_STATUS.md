# NeuroGarden 当前实现状态

## 已完成

### 情绪判断准确性

- 个人习惯样本表、个人基线表、阈值配置表。
- 低负担反馈表。
- 多维情绪状态估计：高压紧张、焦虑紧绷、低落疲惫、疲惫恢复慢、运动干扰、相对稳定。
- 趋势窗口：5 分钟、30 分钟、2 小时。
- 有效提醒率统计。
- 用户轻量标注：累、烦、低落、紧张、没事。
- 近期反馈校准：误报多时降低打扰强度，有效反馈多时提升置信度。

### 被动守护

- 手机端前台服务。
- 本地状态波动通知。
- 通知冷却：同类提醒 15 分钟内不重复，特殊关怀模式冷却更严格。
- 每日通知上限按使用模式生效：自我监测仅本地温和提醒，家庭守护最多 3 次，特殊关怀最多 2 次。
- 数据可信度为 low 时只记录，不升级为强提醒。
- 无障碍输入节奏特征采集：只统计打字速度、删除频率、停顿时长。
- 设置页展示无障碍服务状态、今日输入节奏样本数、最近采集时间和隐私说明。

### AI 关怀

- 后端大模型 API 接入。
- Mock fallback。
- 异常后轻柔对话。
- 对话中二次风险判断。
- 对话摘要入库，不保存原始敏感全文。

### 多元用户支持

- 守护联系人模拟通知。
- 简化关怀模式。
- 隐私与权限说明页。
- 本地习惯记忆清除。

### Wear OS

- Wear 端 Mock 心率。
- Wear 数据 payload 构造。
- 手机端 Wear payload 解析。
- 手机端真实 Wear packet 进入统一风险判断入口。
- Health Services 真实接入状态与权限预留。
- 今日页展示最近心率来源：Mock 或 Real。

### 天气与环境因素

- 天气仓库支持真实网络请求和 Mock 兜底。
- 今日页展示天气因素。
- 风险事件记录异常发生时的天气上下文。
- 网络不可用时不会影响 App 运行。

### 隐私与合规

- 完整隐私与权限说明页。
- 设置页说明健康、无障碍、通知和网络权限用途。
- Agent 数据边界文档：只发送结构化特征和反馈摘要。
- 非医疗诊断声明已补齐。

## 仍需真实接入

### 真实 Wear OS 被动监听

当前已具备手表端手动采集/模拟、Data Layer 同步和手机端接收链路，但还没有完整实现 Health Services Passive Monitoring。下一步需要：

- 使用 Health Services `PassiveMonitoringClient` 或合适的 Measure API。
- 处理心率权限和后台传感器权限。
- 获取真实心率、运动状态、活动类型。
- 通过 Wearable Data Layer 发送到手机。

### 真实通知守护人

当前“通知守护人”仍是模拟。真实实现需要：

- 联系人授权。
- 二次确认。
- 服务端通知或短信能力。
- 通知记录。
- 撤销授权。

### 数据删除细化

当前可清除本地习惯记忆。后续可以拆成：

- 清除关怀记录。
- 清除习惯样本。
- 清除 AI 对话摘要。
- 重置阈值。

### 历史趋势可视化

当前历史页只有摘要。后续可以增加：

- 风险趋势图。
- 高风险时段统计。
- 标注分布。
- 有效提醒率变化。
- 阈值调整记录。

### 真实天气定位

当前天气使用网络查询和 Mock 兜底，但尚未接入系统定位授权。后续可以增加城市选择、定位授权解释和用户手动城市配置。

## 比赛演示建议

1. 启动守护模式。
2. 打开隐私与权限说明，说明系统不采集文本、不做诊断。
3. 在设置页开启 Demo Mode，写入稳定日、轻度波动或夜间异常样本。
4. 在今日页展示天气因素、数据可信度、状态评分和异常事件详情。
5. 在守护页切换自我监测、家庭守护和特殊关怀模式，说明提醒上限差异。
6. 打开输入节奏状态，展示无障碍服务状态、样本数和“不保存输入原文”的说明。
7. 展示异常后的轻柔对话和用户反馈校准。
8. 打开历史页，展示 7 天摘要、反馈统计和趋势闭环。
