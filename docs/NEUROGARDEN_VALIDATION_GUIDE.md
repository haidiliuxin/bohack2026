# NeuroGarden 功能验证文档

本文档用于验证 NeuroGarden 当前版本的核心演示能力：数据采集、风险判断、页面展示、弹窗提醒、聊天关怀、监护反馈、阈值调参和 Wear OS 手表端体征界面。

## 1. 构建与 APK 路径

### 1.1 构建命令

在项目根目录执行：

```bash
./gradlew :app:assembleDebug :wear:assembleDebug
```

Windows PowerShell 可执行：

```powershell
.\gradlew.bat :app:assembleDebug :wear:assembleDebug
```

### 1.2 APK 输出路径

手机端 APK：

```text
app/build/outputs/apk/debug/app-debug.apk
```

Wear OS 手表端 APK：

```text
wear/build/outputs/apk/debug/wear-debug.apk
```

## 2. 手机端基础验证

### 2.1 启动 App

1. 安装并打开 `app-debug.apk`。
2. 进入主界面后，确认底部导航包含：
   - 今日
   - 历史
   - 守护
   - 聊天
   - 设置

通过标准：

- App 可以正常启动。
- 页面无崩溃。
- 底部导航可切换。

## 3. 权限与设置验证

进入“设置”页，检查以下区域：

- Agent 配置
- 数据源状态
- 情绪识别验收
- 输入节奏采集状态
- 设备连接
- 悬浮窗提醒
- Demo Mode

建议开启：

- 无障碍权限：用于采集打字速度、删除频率、停顿时长等统计特征。
- 悬浮窗权限：用于在其他 App 外部直接弹出提醒。
- 通知权限：用于后台提醒。
- 蓝牙权限：用于 Wear OS 连接。

隐私验证点：

- 系统不保存用户输入原文。
- 系统只保存打字速度、删除频率、停顿时长等统计特征。
- Agent 默认只接收结构化特征。
- 文案不出现医学诊断表达。

## 4. 今日页监测闭环验证

### 4.1 Demo Mode 写入数据

进入“设置”页，点击以下任一按钮：

- 模拟稳定一天
- 模拟轻度波动
- 模拟夜间异常
- 模拟监护人确认后调参
- 启动联调演示

推荐优先验证：

```text
启动联调演示
```

预期效果：

- 今日页摘要刷新。
- 今日风险评分曲线有变化。
- 异常事件列表出现新事件。
- 后端/Agent 请求日志可在 Debug 入口看到。
- 若满足弹窗条件，会出现本地提醒或弹窗。

通过标准：

- 今日页能展示风险评分、心率、呼吸、输入速度、删除频率、停顿时长等曲线。
- 异常事件列表不为空。
- 异常详情页可打开。

## 5. 异常事件详情验证

在“今日”页点击异常事件。

检查详情页是否展示：

- 异常时间段
- 风险等级
- Agent 分析原因
- 心率偏离百分比
- 呼吸偏离百分比
- 打字速度偏离百分比
- 删除频率偏离百分比
- 停顿时长偏离百分比
- 是否通知监护人
- 监护人反馈结果

执行反馈按钮：

- 确认异常
- 标记误报
- 已联系本人
- 继续观察
- 提高该类提醒优先级
- 降低该类提醒优先级

通过标准：

- 点击反馈后页面出现调参结果说明。
- `threshold_profiles` 后续会受反馈影响。
- 反馈不会保存用户输入原文。

## 6. 聊天关怀界面验证

进入底部“聊天”页。

当前聊天页应为纯聊天界面，类似豆包式布局：

- 顶部显示 NeuroGarden / 在线陪伴。
- 中间是左右气泡消息。
- 底部是输入框和发送按钮。
- 页面不再展示大块风险仪表盘卡片。

### 6.1 P0 低压关怀动作

检查快捷动作：

- 先停十秒
- 呼吸
- 白噪音
- 暂时安静

点击任一动作后，预期：

- 该动作以用户消息形式进入聊天。
- AI 会继续进行低压陪伴。

### 6.2 P1 偏好反馈

检查反馈按钮：

- 有用
- 没用
- 太打扰
- 少提醒

点击任一反馈后，预期：

- 反馈写入现有反馈链路。
- 后续提醒偏好可被系统学习。

### 6.3 P2 场景化恢复推荐

聊天页应展示“场景恢复推荐”提示和对应快捷动作。

不同场景预期：

- 深夜/夜间：雨声、助眠、短阅读。
- 高删除率/长停顿：整理一句话、先别发送、草稿式表达。
- 早晨压力：拆小任务、一分钟缓冲、今日节奏。
- 运动干扰：运动干扰、补充同步、不用解释。
- 默认场景：轻音乐、散步、安静五分钟。

### 6.4 P3 长期主动调理

聊天页应展示“长期调理建议”提示和对应快捷动作。

预期动作包括：

- 恢复计划
- 周末调理
- 有效方式
- 补数据
- 轻观察
- 明天再看
- 周报
- 偏好
- 轻计划

通过标准：

- 聊天页保留纯聊天形态。
- P0/P1/P2/P3 关怀能力都能通过快捷动作演示。
- 文案不出现“诊断焦虑症”“抑郁症诊断”等表达。

## 7. 后台异常弹窗验证

### 7.1 前置条件

确保已开启：

- 无障碍权限
- 悬浮窗权限
- 通知权限
- 后台守护服务

### 7.2 验证步骤

1. 在 App 设置页启动后台守护。
2. 切到微信、备忘录或其他可输入 App。
3. 快速输入、删除、停顿，制造输入节奏波动。
4. 等待后台服务采样并判断。

预期效果：

- 若达到阈值并满足冷却条件，会在其他 App 上方弹出悬浮提醒。
- 若悬浮窗权限未开，可能只在返回 App 后显示提醒。

通过标准：

- 不需要回到 App 才能看到提醒。
- 弹窗文案为“状态偏离/节律波动/建议确认”等非诊断表述。

## 8. Wear OS 手表端验证

安装并运行：

```text
wear/build/outputs/apk/debug/wear-debug.apk
```

### 8.1 首页验证

手表首页应展示：

- NeuroGarden
- 手机同步状态
- 大号心率圆环
- 心率 bpm
- 状态：稳定 / 观察 / 提醒
- 状态线索
- 置信度
- 数据质量
- 数据来源 Mock / Real
- 呼吸频率
- 运动状态
- 扩展体征：HRV / 体温 / 睡眠
- 判断依据：证据、限制、不确定性

注意：

- 体温、HRV、睡眠当前为预留或待设备支持。
- 不应把预留项展示为真实采集。

### 8.2 手表交互验证

点击：

- 场景：切换 Mock 场景。
- 同步：向手机发送当前体征数据。
- 我还好：记录快速确认状态。
- 稍后：记录稍后提醒状态。
- 呼吸引导：进入手表呼吸光圈页。

通过标准：

- 页面适配圆形屏幕。
- 不是手机 Dashboard 缩小版。
- 同步按钮不崩溃。
- 呼吸引导能显示吸气/呼气，并触发震动节奏。

## 9. 历史页验证

进入“历史”页。

检查最近 7 天统计：

- 每日最高风险分
- 每日异常事件数
- 每日误报数
- 每日确认异常数
- 每日数据可信度

通过标准：

- Demo Mode 写入多日数据后，历史页能显示多日摘要。
- 数据为空时有合理空状态。

## 10. 守护页验证

进入“守护”页。

检查：

- 当前 CareMode 对应文案。
- 自我监测模式隐藏或弱化监护提醒。
- 家庭守护模式显示完整守护设置。
- 特殊关怀模式显示更严格隐私提示和照护说明。
- 今日提醒次数、冷却状态、Agent 状态可见。

通过标准：

- 监护人提醒必须用户授权。
- 不会后台自动拨号或自动发送短信。
- 守护反馈能写入异常事件。

## 11. Agent 审计日志验证

进入 Debug 日志页面。

检查是否展示：

- 最近请求时间
- 触发原因
- HTTP 是否成功
- 后端情绪/状态
- 置信度
- mainReasons
- fallback 原因
- 是否使用缓存
- Prompt 版本或 Agent 版本信息

通过标准：

- 启动联调演示后能看到新的 Agent 审计记录。
- 审计日志不保存用户输入原文。
- 审计日志只保存结构化结果和短摘要。

## 12. 科学与隐私边界验收

全 App 文案必须避免：

- 你有焦虑症
- 你抑郁了
- 临床级情绪识别
- 自残风险预测
- 极端行为预测

推荐表述：

- 状态偏离
- 节律波动
- 建议确认
- 重点关注
- 守护提醒
- 照护确认

通过标准：

- 所有页面都强调非医学诊断。
- 被动采集不读取用户输入原文。
- 聊天页只处理用户主动输入的当前对话内容。

## 13. 最终演示闭环

推荐答辩演示顺序：

1. 设置页点击“启动联调演示”。
2. 今日页查看风险曲线和异常事件。
3. 打开异常详情，展示 Agent 分析和偏离百分比。
4. 点击监护人反馈，例如“确认异常”。
5. 展示阈值调参结果。
6. 进入聊天页，展示低压关怀动作。
7. 展示 P2 场景恢复推荐。
8. 展示 P3 长期调理建议。
9. 打开 Wear OS 端，展示心率圆环、状态线索、证据限制和呼吸引导。
10. 回到历史页，展示最近 7 天统计。

闭环说明：

```text
结构化采集
-> 本地清洗与风险判断
-> Agent 结构化分析
-> 今日页与异常事件展示
-> 弹窗/聊天关怀
-> 用户或监护人反馈
-> 阈值与偏好调参
-> 历史趋势和长期调理建议
```

## 14. 当前已知限制

- 真实体温、HRV、睡眠数据目前未完整接入，手表端仅做预留展示。
- 呼吸数据在无真实设备时为估算或 Mock。
- Agent 输出用于结构化辅助解释，不作为最终诊断。
- 当前准确率更多体现为用户反馈接受率，不是医学实验准确率。
- Debug APK 未做正式签名，不用于生产发布。
